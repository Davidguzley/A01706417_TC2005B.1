Laboratorio 1: Introducción a las aplicaciones web, HTML5 y ciclo de vida de los sistemas de información

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    • El internet es la red de computadoras y www es la colección de páginas que estan en la red. Necesitas internet para acceder a la web.

  2. ¿Cuáles son las partes de un URL?
    • Protocolo/sub dominio/dominio/ruta/parametros

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • GET: Solicita recursos especificados.
    • HEAD: Provee información general o dicho de otro modo metadatos.
    • POST: Envía recursos.
    • PUT: Crea un nuevo elemento o reemplaza una representación del elemento de destino con los datos de la petición.
    • PATCH: Modificaciones a un recurso.
    • DELETE: Borra un recurso.
    Referencia: www.freecodecamp.org

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    • POST, ya que solo se manda lo solicitado por una pagina.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    • GET, ya se obtiene los recursos de la pagina.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
    • Significa que la página ha cargado de forma correcta.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
    • No es su responsabilidad porque el usuario a intentado entrar en donde no debia.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
    • En este caso si es su responsabilidad, ya que el error significa que el error ha sucedido al intentar acceder al servidor y debe ser arreglado.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
    • Que ya no son utilizadas en HTML5 ya que sus funciones se manejan mejor con CSS. Elementos desaprobados en HTML5: applet, dir, strike, etc.

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    • HTML5 continúa en progreso.
    • Sintaxis simplificada
    • Deesaparecen frame, center y big.
    • Estructura sin uso de div id= "algo".
    Referencia: www.sutori.com

  11. ¿Qué componentes de estructura y estilo tiene una tabla?
    • table: crear tabla.
    • thead: Delimita el encabezado de la tabla.
    • tbody: Delimita el cuerpo de la tabla.
    • tfoot: Delimita el pie de página.
    • caption: título.
    • col: columna.
    • td: datos de una celda.
    • tr: Fila.
    • th: celda de cabecera.
    Referencia: jorgesanchez.net

  12. ¿Cuáles son los principales controles de una forma HTML5?
    • form: Formulario.
    • button: Boton.
    • label: Titulo de control de un formulario.
    • datalist: Representa un conjunto de opciones predefinidas para otros controles.
    • option:Representa una opción en un elemento select, o una sugerencia de un elemento datalist.
    • progress: Representa el progreso de finalización de una tarea.
    • input: Entrada del usuario.
    • output: Resultado de un calculo.
    • select: Representa un control que permite la selección entre un conjunto de opciones.
    Referencia: www.w3.org

  13. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas? Puedes utilizar la siguiente página para descubrirlo: http://html5test.com/ (Al responder la pregunta recuerda poner el navegador que utilizas)
    • Estoy usando Google Chrome tiene una puntuacion de 476/555 con HTML5.

  14. ¿Cuál es el ciclo de vida de los sistemas de información? y ¿Cuál es el ciclo de desarrollo de sistemas de información?
    • El ciclo de vida de un sistema de información es un enfoque por fases del análisis y diseño que sostiene que los sistemas son desarrollados de mejor manera mediante el uso de un ciclo especifico de actividades del analista y del usuario.
    • El ciclo de desarrolo de sistemas de informacion son fases para desarrollar un sistema de información, en el cual permita ir planteando soluciones a los problemas que puedan ir surgiendo en dicho proceso.
    Referencia: VIEWNEXT


Laboratorio 3: CSS

  1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
    • Que solo se utilice cuando es necesario ya que la instruccion declarada no podra ser sobreescrita , dicho de otro modo esta permanera igual siempre.

  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
    • Porque esta puede afectar distintos aspectos de la pagina como su estetica o rendimiento. Ademas de claramente los derechos de autor de las imagenes.

  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
    • Que siempre se procure elegir unidades relativas como el %, ya que esta se adaptara dependiendo del tamaño de la ventana del dispocitivo o navegador donde se visualice la pagina.

  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
    • Debido a que las hojas de estilos son cargadas a las pagias y entre menos sea el tamaño de estas el rendimiento sera mucho mejor. Aunque una vez minimizadas sera muy complicado modificar el CSS.
